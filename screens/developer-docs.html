Developer Documentation

Logic, flows, and screen-by-screen behaviour for Content Manager by OLX. Use this as the single source of truth when implementing.

Product Overview

Content Manager by OLX is a web tool that allows sellers on competitor platforms (AVTO.pro, PROM.ua) to import their listings to OLX. The user uploads a file exported from a competitor platform, maps any required fields, and the system publishes the listings to their connected OLX account.

  • Single-page app — all screens live in one HTML shell, loaded dynamically from screens/.
  • Bilingual — Ukrainian (UA) and English (EN). Switching is global and instant. Text lives in data-ua / data-en attributes; applyLang() applies them on every screen change.
  • Design tokens: --olx-charcoal (#044142), --olx-teal-dark (#0EA9A0), --olx-teal-light (#CAFCFA), --olx-yellow-dark (#FFD13C).
  • Bootstrap 5.3 for layout and utilities; custom component classes in assets/styles.css.
  • Comment bubbles (.cb) throughout the prototype expose design notes via a modal. Toggle visibility with the global comments button.
Global Navigation & Header
  • Logo click → always navigates to Import History (screen 12).
  • Header links: "Import History" → screen 12; "New Import" → screen 6 (import wizard).
  • "My Profile" dropdown shows the user's registered email at the top (read-only), then Settings and Sign Out.
  • Header is rendered via renderAppHeader(screenId) and mounted into any element with data-component="app-header".
  • Auth screens (Register, Login, Forgot Password, Email Verification) show the split-panel layout (left brand panel + right form). They do not show the app header.
  • Post-auth screens (onboarding, import, history, settings) show the full app header.
Screen 1 — Registration
  • Fields: Email, Password (with strength meter), Confirm Password.
  • Password strength bar: 4 segments, colour-coded. Rules: min 8 chars, at least one uppercase, one lowercase, one digit.
  • On success → navigates to Email Verification (screen 3).
  • Left panel: brand copy + 3 feature bullets.
Screen 2 — Login
  • Fields: Email, Password (toggle visibility).
  • Forgot Password link → screen 10.
  • On success → navigates to Import History (screen 12).
Screen 10 — Forgot Password
  • Single email field. On submit → shows inline success state (no separate screen): "Check your inbox" message replaces the form.
  • Back to Login link always visible.
Screen 3 — Email Verification
  • Shows the registered email address inline: "Email sent to user@example.com" — on one line.
  • 60-second countdown before the "Resend" button becomes active. Countdown and button are centred; the rest of the text is left-aligned.
  • On "Confirm" (simulated) → navigates to Onboarding OLX Sync (screen 4).
Screen 4 — Onboarding: OLX Sync (Step 1 of 2)
  • Label above stepper: "Getting Started". Numbered stepper (1 → 2), centred. Step 2 is locked until OAuth completes.
  • Explains why OLX API access is needed. CTA: "Grant OLX Access" → opens OAuth simulation modal.
  • OAuth modal simulates redirect to OLX.ua. On "Allow Access" → success state shows connected email + edit (pencil) button inline. "Next →" button appears. Step 2 dot unlocks.
  • Pencil button → re-opens OAuth modal to change the connected account.
  • Step dots are clickable only when their step is reached. Step 2 cannot be accessed before OAuth completes.
  • On "Next →" → screen 5.
Screen 5 — Onboarding: Contacts (Step 2 of 2)
  • 3 fields only: Contact Name / Company, Location (searchable dropdown), Phone.
  • Location dropdown: Bootstrap dropdown with a live search input. 33 Ukrainian cities pre-loaded. Selecting a city closes the dropdown and updates the label.
  • Phone: single text field, supports multiple numbers comma-separated. Hint: "You can specify multiple numbers separated by comma."
  • All fields pre-filled with data pulled from the OLX account via API (design note comment on field).
  • Buttons: Ghost "← Back" (→ screen 4), Primary "Done" → validates all fields → navigates to Import History (screen 12).
  • Step 1 dot is clickable (marked done with checkmark), Step 2 is active.
Screen 12 — Import History (Main Dashboard)

This is the main screen of the application. Users land here after onboarding and after every import.

Two states
  • Empty state: shown when user has no imports yet. Large inbox icon, descriptive text, no table. "+ New Import" button is always visible in the header area.
  • Active / populated state: shown after first import starts. Active import card appears at the top, history table below.
Active Import Card
  • Left border teal accent. Shows: file name, platform badge chip, "In progress" badge (teal).
  • Processing counter: "Processing: N / 146 listings".
  • Live counters: Published (green) and Failed (red), updated every ~70ms during simulation.
  • Progress bar (right side) with percentage and estimated time remaining (calculated from elapsed rate).
  • The active import does not appear in the history table — it is added to the table only when complete.
Email notification info block
  • .status-block.info below the active card: "Once all listings are processed, we will send you a confirmation email."
History Table columns
  • File, Platform (badge chip), Status (badge), Total, Success, Failed, Added at (date + time on two lines), Actions.
  • Status badges: .badge-success "Completed", .badge-status "In queue". Active imports are never in this table.
  • Actions: "Fix errors" button (.btn-sm-outline with pencil icon) — shown only for completed rows with failed > 0. Rows with 0 failures or "In queue" show "—".
Screen 6 — New Import Wizard

3-step wizard. Stepper is always pinned at the top. Steps unlock progressively.

Step 1 — Platform
  • Two platform cards: PROM.ua, AVTO.pro — each with a .badge-platform chip. Click to select (card gets .selected class).
  • Buttons: Ghost "Cancel" (→ history screen), Primary "Next →".
  • "Next" validates that a platform is selected; shows error toast if not.
  • Selecting a platform unlocks Step 2 dot in the stepper.
Step 2 — File Upload
  • Header shows: "Platform: [badge chip]".
  • .status-block.info block: "First download your listings from [Platform]." with "View instructions →" link that opens the slide panel.
  • Instructions slide panel: opens from the right (400px), platform-specific step-by-step guide.
  • Drop zone: click or drag to "upload". On file attached — zone gets .has-file class (teal border + teal-light background), shows file name + "File contains: 146 listings".
  • Checkbox: "Skip listings without photos" (checked by default).
  • AVTO.pro only: extra fields appear — Currency select (UAH / USD / EUR, default USD), Company Description textarea (required, min 40 chars).
  • Buttons: Ghost "← Back", Primary "Next →". "Next" validates: file attached; if AVTO.pro — description not empty and ≥ 40 chars.
  • Attaching a file unlocks Step 3 dot.
Step 3 — Confirm
  • Summary card: Platform (badge), File name + listing count, Photos setting, Currency (AVTO.pro only), Description (AVTO.pro only).
  • Contact details collapsible block: shows Name · Location · Phone in one collapsed line. Expand to edit all three fields inline. Changes apply to this import only — profile data is not modified. Summary line updates live as user types.
  • Buttons: Ghost "← Back", Primary "Import to OLX →" → calls startImport().
  • startImport() sets importHistoryHasActive = true and navigates to screen 12 (history) which starts live progress animation.
Stepper behaviour
  • Active step: teal circle with number. Done step: teal circle with checkmark icon. Future step: grey circle.
  • Completed step dots are clickable (navigate back). Future step dots are not clickable.
  • Navigating back via a stepper dot does not reset data already entered.
  • Resetting the wizard (resetImportWizard()) clears platform selection, file, and AVTO.pro fields.
Screen 8 — Failed Listings

Accessible from "Fix errors" button in the history table. Shows only listings that failed — not all OLX account listings.

Import meta block
  • File name, platform badge, status badge (.badge-success).
  • Metadata row: Import started (date + time), Completed (date + time + duration), Total, Success, Failed.
Table
  • Columns: Photo (48×48 thumbnail), Title, Error reason (.err-badge), Price, Actions.
  • Pagination: select box (10 / 50 / 100 per page). Prototype — UI only, no actual data slicing.
Row actions
  • "Edit" (primary): opens slide-in panel with editable fields. Save button: "Save and import" → attempts re-import. On success: toast + row fades out and is removed. Count in meta block decrements.
  • "Retry" (outlined): attempts re-activation without editing. Behaviour depends on error type (see below).
Error scenarios (prototype)
  • Row 0 — "Category not specified": Retry → row stays, error toast shown. Edit → success, row removed.
  • Row 1 — "Title too long": First save → a second error appears ("Invalid price — below the minimum allowed"), row stays with updated error badge. Second save → success, row removed. Demonstrates multi-error flow.
  • Row 2 — "Internal server error (500)": Retry → success without editing, row removed. This represents transient backend errors that resolve on retry.
  • Row 3 — "Missing photo": Retry → row stays, error toast. Edit → success, row removed.
  • When all rows are removed → table is hidden, success empty state shown.
Edit slide panel
  • Error reason shown in red alert at the top.
  • Fields: Category, Part type, Brand, Manufacturer, Part code, Condition (toggle buttons), Listing title (with 0/70 char counter), Description.
  • Footer: Primary "Save and import", Ghost "Cancel".
Screen 9 — Settings

Left sidebar nav + content panel. Four tabs.

OLX Sync
  • Shows connection status: "Connected: [email]" in .olx-sync-success block.
  • Pencil button next to it → re-opens OAuth modal to switch account.
  • Descriptive text adapted for already-connected state: explains how to change the account, not how to connect.
Contact Details
  • Same 3 fields as onboarding step 2: Contact name, Location (searchable dropdown), Phone.
  • Pre-filled from user profile. These are the default values used across all imports.
  • Save → toast confirmation. Changes are global (affect all future imports).
Company Description
  • Single textarea. Used as listing description for AVTO.pro imports (AVTO.pro files contain no descriptions).
  • Min 40 characters. Live char counter turns red below threshold. Save validates minimum.
  • This is the default; can be overridden per-import on the wizard Step 3 confirmation screen.
Change Password
  • Current password + New password (with strength meter). Save → toast confirmation.
Contact Data Hierarchy
  • Level 1 — Profile defaults: set in Settings → Contact Details. Used for all imports.
  • Level 2 — Per-import override: editable on Step 3 of the import wizard. Applies only to that specific import request. Does not affect profile defaults.
  • The wizard pre-fills Level 2 fields from Level 1 on each new import.
AVTO.pro — Platform-specific Logic
  • AVTO.pro export files do not include listing descriptions or currency information.
  • Currency: user selects file currency in Step 2 (UAH / USD / EUR, default USD). Shown in Step 3 summary. Used for price conversion on import.
  • Description: required text field in Step 2 (min 40 chars). Applied to all imported listings from this file. Can be pre-set globally in Settings → Company Description.
  • Both fields are shown only when AVTO.pro is selected. Hidden for PROM.ua.
Toast Notifications
  • API: toast(message, type). Types: default (success/info), 'error'.
  • Style: white card, 4px left border (teal for default, red for error), drop shadow. Appears top-centre. Auto-dismisses after 5 seconds.
  • Multiple toasts stack vertically.
