# Catalog by OLX — UI Prototype Instructions

## Завдання

Створити **інтерактивний HTML-прототип** (один файл `prototype.html`) який:
- Відкривається локально в браузері
- Має **9 клікабельних екранів** з переходами між ними
- Має **панель навігації** (кнопка внизу справа) зі списком всіх екранів
- Має **перемикач мов UA / EN** (внизу зліва)
- Має **іконки-коментарі** (помаранчеві кружечки з іконкою чату) на кожному місці де є зміни/зауваження — при кліку показують попап з текстом зауваження (двома мовами)
- Форми **симулюють роботу** (валідація, переходи між екранами)

## Технічний стек прототипу

- Bootstrap 5 CDN
- Bootstrap Icons CDN
- Google Fonts: Roboto
- Чистий JavaScript (без фреймворків)
- Один HTML файл

## Брендові кольори OLX

```css
--olx-charcoal: #044142;   /* хедер, темні кнопки */
--olx-teal: #1AD8D2;
--olx-teal-dark: #0EA9A0;  /* акценти, активні стани */
--olx-teal-light: #CAFCFA;
--olx-yellow-dark: #FFD13C; /* крапка в логотипі */
--olx-yellow: #FFDD73;
```

## ЕКРАН 1 — Реєстрація (дефолтний екран)

### Макет
- **Ліва панель (40%)**: темний фон `#044142`, повторювана сітка білих іконок автозапчастин (Bootstrap Icons: wrench, gear, tools, car, truck, speedometer тощо), поверх іконок — текст про сервіс
- **Права панель (60%)**: біла, форма реєстрації

### Ліва панель — текст
```
catalog.
by OLX

Перенесіть свої оголошення на OLX

✓ Ми опублікуємо ваші оголошення на OLX
✓ Слідкуйте за прогресом в особистому кабінеті
```

### Форма реєстрації
- Заголовок: "Створити обліковий запис"
- Підзаголовок: "Вже є акаунт? [Увійти]" (посилання → Екран 2)
- Поле Email
- **⚠️ Інформаційний блок** (блакитний) ПІД полем Email:
  > "Обліковий запис Content Manager не пов'язаний з обліковим записом OLX. Якщо ви користуєтесь сервісом вперше — [зареєструйтесь]."
- Поле Пароль (з іконкою показати/сховати)
- **Прогрес-бар пароля** (4 секції, реально працює):
  - Умови: довжина ≥8, велика літера, мала літера, цифра
  - 1/4 = червоний, 2/4 = помаранчевий, 3/4 = жовтий, 4/4 = зелений
  - Підпис: "Мінімум 8 символів: велика, мала літера та цифра"
- Текст про умови використання (дрібний)
- Кнопка "Зареєструватися" (темна, 100% ширина) → Екран 3
- Кнопка "Увійти через Google" (з логотипом Google)

### Коментарі на цьому екрані
1. На лівій панелі: "Зміна: Додано опис сервісу. Раніше тут були лише іконки. Тепер користувач одразу розуміє що це за сервіс — міграція запчастин з AVTO.pro/PROM.ua на OLX."
2. На інфо-блоці: "Зміна: Блок перенесено ПІД поле Email (не над формою). Додано посилання на реєстрацію всередині блоку."
3. На прогрес-барі пароля: "Баг виправлено: прогрес-бар не фарбувався в правильний колір навіть коли всі умови виконані. Реалізована правильна логіка: 4 умови → 4 кольори."
4. На заголовку: "Зміна: Дефолтна сторінка — реєстрація, а не логін (після уточнення терміну дії токена)."

## ЕКРАН 4 — Онбординг: Синхронізація OLX

**Цей екран показується ТІЛЬКИ перший раз після реєстрації.**

### Макет
- Хедер застосунку (темний)
- Індикатор прогресу: Крок 1 з 2 (кружечки: 1=активний, 2=неактивний)
- Картка по центру

### Зміст картки
- Заголовок: "Синхронізація з OLX"
- **Опис нормальним читабельним шрифтом**: "Натисніть кнопку нижче — вас перекине на OLX.ua. Увійдіть у свій акаунт та дайте додатку доступ."
- Кнопка "Синхронізувати" → відкривається модальне вікно-симуляція OAuth

**Симуляція OAuth модал:**
- Логотип OLX
- Текст: "Симуляція: Ви переходите на OLX.ua для авторизації. Натисніть 'Дозволити доступ' щоб продовжити."
- Кнопка "Дозволити доступ" → закриває модал, показує успішний стан
- Кнопка "Скасувати"

**Після успішної синхронізації:**
- Зелений блок: "✓ Синхронізовано з акаунтом OLX: **mishaborys@gmail.com**" (або телефон якщо email відсутній)
- Кнопка змінюється на "Синхронізувати інший акаунт"
- З'являється кнопка "Далі →" → Екран 5

**Важливо:** Якщо користувач НЕ синхронізував — кнопка "Далі" недоступна (disabled).

### Коментарі
1. "Зміна: Текст опису зроблено нормальним читабельним шрифтом. Раніше він був майже невидимий."
2. "Важливо: Після синхронізації показуємо email АБО телефон (деякі акаунти OLX.ua мають тільки телефон, без email)."
3. "Зміна: Кнопка після синхронізації змінюється на 'Синхронізувати інший акаунт'. Ніяких зайвих описів після успіху — тільки статус і кнопка 'Далі'."
4. "Зміна: Ці 2 кроки онбордингу повністю відокремлені від флоу імпорту. Раніше вони були частиною 5-кроківного wizard'у — це неправильно."

## ЕКРАН 5 — Онбординг: Контактні дані

**Цей екран показується ТІЛЬКИ перший раз після реєстрації.**

### Макет
- Хедер застосунку
- Індикатор: Крок 2 з 2 (кружечок 1 = виконано ✓, кружечок 2 = активний)
- Картка

### Зміст картки
- Заголовок: "Контактні дані"
- Підзаголовок: "Ці дані будуть використані для публікації оголошень на OLX. Ми підтягнули їх автоматично — ви можете змінити."
- **Поля (підтягуються з OLX API, але редаговані):**
  - "Контактна особа або назва компанії" (text, обов'язкове)
  - "Область" (select, обов'язкове) — приклади: Київська, Харківська...
  - "Населений пункт" (select, обов'язкове) — залежить від Області
  - "Район міста" (select) — **з'являється ТІЛЬКИ якщо місто має райони (наприклад Київ)**. Для інших міст — не показувати.
  - "Телефон" (text, НЕ number, обов'язкове) — з кодом країни (+380), підтримує кілька номерів через кому (макс 3)
  - "Телефон 2" (text) — **з'являється тільки якщо заповнений Телефон 1**
  - "Телефон 3" (text) — **з'являється тільки якщо заповнений Телефон 2**
  - Тільки перший телефон обов'язковий

- **Кнопка "Далі →"** (без окремої кнопки "Зберегти") — валідує і зберігає → Екран 6

### Коментарі
1. "Зміна: Дані автоматично підтягуються з OLX API. Поля передзаповнені але редаговані."
2. "Зміна: Поле 'Район міста' з'являється ТІЛЬКИ для міст що мають райони (напр. Київ). Для інших — не показувати."
3. "Зміна: Телефон — текстове поле (не числове). Без стрілок вгору/вниз. Підтримує кілька номерів через кому. Телефон 2 і 3 з'являються умовно."
4. "Зміна: Кнопки 'Зберегти інформацію' НЕ існує. Кнопка 'Далі' одночасно валідує і зберігає."

### ТАБ: Імпортувати оголошення

#### Вибір платформи
Два великі картки-таби (при першому відкритті — жоден не вибраний):
```
┌──────────────────┐  ┌──────────────────┐
│   PROM.ua        │  │   AVTO.pro       │
│ Імпорт з Prom.ua │  │ Імпорт з Avto.pro│

```
Після вибору — картка підсвічується (teal border + teal-light background).
Вибір зберігається в акаунті як дефолт.

#### Після вибору платформи — показати:

**Інфо-банер (жовтий):**
> ⚠️ Щоб імпортувати оголошення, спочатку завантажте файл з **[назва платформи]**.
> Не знаєте як? → **[Переглянути інструкцію]** (відкриває модальне вікно)

**Велика зона завантаження файлу:**
```
┌──────────────────────────────────────────┐
│                                          │
│     📁  (велика іконка)                  │
│   Перетягніть файл або оберіть вручну    │
│       .xlsx, .xls                        │
│                                          │
│      [ Обрати файл ]                     │
└──────────────────────────────────────────┘
```
Після вибору файлу зона змінюється:
```
┌──────────────────────────────────────────┐
│  ✓ avto_export.xlsx                      │
│  Файл містить: 146 оголошень             │
└──────────────────────────────────────────┘
```

**Чекбокс** (за замовчуванням увімкнений):
☑ Не завантажувати оголошення без фото

**Тільки для AVTO.pro** — після вибору файлу з'являються додаткові поля:

Опис оголошень (обов'язково):

(Цих даних немає у файлі AVTO.pro)
```

**Кнопка "Імпортувати на OLX →"** (праворуч) → Екран 7

#### Модальні вікна з інструкціями

**Інструкція AVTO.pro:**
1. Авторизуйтесь на сайті avto.pro
2. У боковому меню оберіть розділ **Склади**
3. Відкрийте один зі своїх складів
4. У правому верхньому куті натисніть **Експортувати склад**
5. Оберіть формат **Excel**
6. Завантажте файл на пристрій
7. Прикріпіть файл у нашій системі

⚠️ Важливо: Не змінюйте структуру файлу. Не переставляйте та не видаляйте колонки. Рядки можна видаляти.

**Інструкція PROM.ua:**
1. Авторизуйтесь на сайті prom.ua
2. Товари та послуги → **Товари**
3. Натисніть кнопку **Експорт**
4. Оберіть **Excel (.xlsx)**, мову полів — українська
5. Натисніть **Експортувати**
6. Ви отримаєте листа з посиланням для завантаження
7. Завантажте файл та прикріпіть у нашій системі

⚠️ Важливо: те саме що для AVTO.pro

## ЕКРАН 8 — Зафейлені оголошення

### Хедер + кнопка "← Назад до кабінету"

Імпорт: prom_export_01042026.xlsx

### Таблиця (ТІЛЬКИ зафейлені)
Колонки: Фото | Заголовок | Причина помилки | Ціна | Дії

Приклад рядків:
```

```

### Бокова панель редагування (slide-in з правого боку)
Відкривається при кліку "Редагувати":

- Заголовок панелі: "Редагування оголошення"
- **Червоний блок з причиною помилки**: "⚠️ Причина: Не вказана категорія"
- Поля:
  - Категорія (select)
  - Тип частини (select)
  - Марка (select)
  - Виробник (text)
  - Код частини (text)
  - Стан: [Вживаний] [Новий] (toggle)

- Помаранчевий кружечок (#FF5722) з іконкою bi-chat-fill
- При кліку — показується модальний попап з текстом

function checkPassword(password) {
  const conditions = [
    password.length >= 8,
    /[A-Z]/.test(password),
    /[a-z]/.test(password),
    /[0-9]/.test(password)
  ];
  const score = conditions.filter(Boolean).length; // 0-4
  // 1 = червоний, 2 = помаранчевий, 3 = жовтий, 4 = зелений
}
```
